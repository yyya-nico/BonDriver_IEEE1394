# BonDriver_IEEE1394 使用ガイド

## 概要

BonDriver_IEEE1394は、IEEE 1394（FireWire/i.LINK）接続のD-VHSなどのHDV機器からTSストリームを取得し、テープ操作を行うためのBonDriverです。

## インストール

### 1. ビルド済みDLLの配置

`BonDriver_IEEE1394.dll`を使用するアプリケーションのBonDriverフォルダにコピーします。

例：
```
C:\Program Files\TVTest\BonDriver\BonDriver_IEEE1394.dll
```

### 2. 設定ファイル（オプション）

同じフォルダに`BonDriver_IEEE1394.ini`を配置することで、動作をカスタマイズできます。

```
C:\Program Files\TVTest\BonDriver\BonDriver_IEEE1394.ini
```

## 対応デバイス

以下のIEEE 1394デバイスに対応しています：

- D-VHSデッキ
- HDVカメラ
- その他のAV/C準拠テープレコーダー/プレイヤー

## 接続方法

### 1. ハードウェア接続

1. IEEE 1394ケーブル（FireWireケーブル）でPCとDV機器を接続
2. HDV機器の電源を入れる
3. テープを挿入（再生する場合）

### 2. Windowsでの確認

1. デバイスマネージャーを開く
2. 「イメージングデバイス」または「サウンド、ビデオ、およびゲームコントローラー」の下にデバイスが表示されることを確認

### 3. アプリケーションでの使用

#### TVTest での使用例

1. TVTestを起動
2. メニュー → 設定 → BonDriverの設定
3. BonDriver_IEEE1394を選択
4. チューナーを開く

#### チャンネル操作

- **チャンネル 0**: 停止
- **チャンネル 1以上**: 再生開始

## 設定ファイル (BonDriver_IEEE1394.ini)

```ini
[Device]
; デバイス名のフィルタ（部分一致）
; 空の場合は最初に見つかったIEEE 1394デバイスを使用
DeviceNameFilter=

; デバイスインデックス（0から開始）
; -1の場合は最初に見つかったデバイスを使用
DeviceIndex=-1

[Buffer]
; TSストリームキューの最大サイズ
MaxQueueSize=256

; TSパケットサイズ（通常は188バイト）
PacketSize=188

[Transport]
; デバイスオープン時に自動再生するか
AutoPlay=false

; 再生開始時の待機時間（ミリ秒）
PlayStartDelay=1000

; コマンド実行後の待機時間（ミリ秒）
CommandDelay=100

[Debug]
; デバッグログを有効にする
EnableLog=false

; ログファイルパス
LogFilePath=BonDriver_IEEE1394.log
```

### 設定項目の説明

#### [Device]

- **DeviceNameFilter**: 特定のデバイスのみを使用する場合、デバイス名の一部を指定
  - 例: `DeviceNameFilter=Sony` → Sonyを含むデバイスのみ
  - 空の場合は最初に見つかったデバイスを使用

- **DeviceIndex**: 複数のIEEE 1394デバイスがある場合、どのデバイスを使うか指定
  - 0: 最初のデバイス
  - 1: 2番目のデバイス
  - -1: 自動選択

#### [Buffer]

- **MaxQueueSize**: TSパケットのキューサイズ（デフォルト: 256）
  - 大きくすると遅延が増えるが、パケットロスが減る
  - 小さくすると遅延が減るが、パケットロスが増える可能性

- **PacketSize**: TSパケットのサイズ（通常は188バイト）

#### [Transport]

- **AutoPlay**: デバイスオープン時に自動的に再生を開始するか
  - `true`: 自動再生
  - `false`: 手動で再生開始が必要

- **PlayStartDelay**: 再生開始時の待機時間（ミリ秒）
  - デバイスの準備に時間がかかる場合は増やす

- **CommandDelay**: コマンド実行後の待機時間（ミリ秒）

#### [Debug]

- **EnableLog**: デバッグログを有効にする
- **LogFilePath**: ログファイルの保存先

## トラブルシューティング

### デバイスが見つからない

**症状**: `IEEE 1394デバイスを開けませんでした` と表示される

**対処法**:
1. IEEE 1394ケーブルが正しく接続されているか確認
2. HDV機器の電源が入っているか確認
3. Windowsのデバイスマネージャーでデバイスが認識されているか確認
4. ドライバーが正しくインストールされているか確認

### TSストリームが受信できない

**症状**: パケットが受信されない、または受信が途切れる

**対処法**:
1. テープが正しく挿入されているか確認
2. テープが破損していないか確認
3. デバイスが再生状態になっているか確認（`T`キーで状態確認）
4. 設定ファイルで`MaxQueueSize`を増やす

### 操作コマンドが効かない

**症状**: 早送りや巻き戻しができない

**対処法**:
1. デバイスがAV/C準拠か確認
2. デバイスのマニュアルで対応コマンドを確認
3. 一部の機能はデバイス側でサポートされていない可能性

### 映像が乱れる

**対処法**:
1. IEEE 1394ケーブルを交換
2. ケーブルの長さを短くする（4.5m以下推奨）
3. PCのIEEE 1394ポートを変更
4. 他のIEEE 1394デバイスを外す

### パフォーマンスの問題

**症状**: パケットロスやコマ落ちが発生

**対処法**:
1. CPUの使用率を確認
2. 他のアプリケーションを終了
3. 設定ファイルで`MaxQueueSize`を増やす
4. ウイルス対策ソフトを一時的に無効化

## よくある質問（FAQ）

### Q1: D-VHSデッキでの録画TSストリームは取得できますか？

**A**: はい、D-VHSデッキに録画されたTSストリームを取得できます。ただし、デバイスがIEEE 1394経由でのTS出力に対応している必要があります。

### Q2: 複数のデバイスを同時に使用できますか？

**A**: 現在のバージョンでは1台のデバイスのみサポートしています。複数台使用する場合は、複数のBonDriverインスタンスを設定ファイルで個別に指定してください。

### Q3: リアルタイム録画は可能ですか？

**A**: はい、TVTestなどの対応アプリケーションでリアルタイム録画が可能です。

### Q4: どのくらいのビットレートまで対応していますか？

**A**: IEEE 1394の帯域（最大400Mbps）まで対応していますが、実際には25Mbps程度のTSストリームが一般的です。

### Q5: タイムコードの取得はできますか？

**A**: はい、一部のデバイスではタイムコードの取得が可能です。ただし、すべてのデバイスが対応しているわけではありません。タイムコード取得には、デバイスがAV/C準拠である必要があります。

### Q6: Linuxでは使えますか？

**A**: このBonDriverはWindows専用です。Linux環境では別のソリューションが必要です。

## サポート対象外の機能

以下の機能は現在サポートされていません：

- 複数デバイスの同時使用
- HDVカメラの撮影制御
- デバイスの詳細設定（ビットレート、画質など）
- テープの自動頭出し

## 技術仕様

- **対応OS**: Windows 10/11 (x64)
- **インターフェース**: IEEE 1394 (FireWire 400/800)
- **ストリーム形式**: MPEG-2 Transport Stream
- **パケットサイズ**: 188バイト
- **バッファサイズ**: 可変（デフォルト256パケット）
